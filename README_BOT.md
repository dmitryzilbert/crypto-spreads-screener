# Telegram бот для MEXC MM Screener

Бот запускается отдельным процессом и читает события из `events.jsonl`, не влияя на работу скринера.

## Почему python-telegram-bot

- Поддерживает `asyncio` из коробки (нет потоков), что соответствует требованиям проекта.
- Минимальные зависимости и простой API для polling.
- Хорошо документирована и позволяет быстро собрать MVP без сложного кода.

## Что он делает

- Tail-читает `events.jsonl` и отправляет уведомления `go_on` (и `go_off`, если включено в конфиге) с анти-спамом по символу.
- Обрабатывает команды `/top [N]`, `/status`, `/mute SYMBOL`, `/unmute SYMBOL`, `/start`.
- Если бот падает или нет соединения — скринер продолжает писать события как обычно.

## Конфиг

Смотрите `config.telegram.yaml` — токен, чат, путь к events.jsonl, лимиты отправки и настройки команд.

## Запуск

```bash
python -m pip install -r requirements-bot.txt
python bot_main.py --config config.telegram.yaml
```
